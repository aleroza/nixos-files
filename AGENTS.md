# AGENTS.md — Инструкции для LLM агентов

## Общие правила

При работе с Nix-конфигурацией (`*.nix` файлы) **всегда** выполняйте валидацию после внесения изменений.

## Валидация Nix-конфигов

После любых изменений в файлах `*.nix` (включая `configuration.nix`, `home.nix`, `flake.nix` и т.д.) необходимо:

1. **Синтаксическая проверка** — запустить `nixfmt` для форматирования и проверки синтаксиса:
   ```bash
   nixfmt <файл>
   ```

2. **Валидация конфигурации** — запустить `nix-eval` или `nix-build` для проверки корректности:
   ```bash
   nixos-rebuild dry-run --flake .#<hostname>
   ```
   (замените `<hostname>` на актуальное имя хоста, например `nixos` или `openclaw`)

3. **Проверка flake** — если изменялся `flake.nix`:
   ```bash
   nix flake check
   ```

## Когда НЕ нужно валидировать

- Изменения в не-Nix файлах (README.md, .gitignore и т.п.)
- Изменения в документации или комментариях внутри Nix-файлов

## Важно

- Если валидация обнаружила ошибки — исправьте их перед завершением работы
- Сообщайте о результатах валидации в `attempt_completion`
